# Sistema UX/UI

## Concepto: Mesa de Proyecto

La pantalla de escritorio utiliza tres áreas:

```text
┌──────────────┬─────────────────────────────┬────────────────────┐
│ Capas        │ Ambiente central            │ Ficha viva         │
│ del proyecto │ foto/plano, medidas, color  │ cantidad/stock/$   │
└──────────────┴─────────────────────────────┴────────────────────┘
```

## Pantallas principales

1. Inicio por intención: subir foto, medir, describir o abrir proyecto.
2. Brief del ambiente.
3. Mesa de proyecto.
4. Comparación económica/recomendada/premium.
5. Cotización editorial.
6. Compra, recojo o despacho.
7. Mis proyectos y recompra.
8. Vista profesional multiambiente.
9. Administración por dominios.

## Componentes distintivos

- Ambiente Canvas.
- Regla Viva.
- Pila de Color.
- Medidor de Cobertura.
- Ensamblador de Envases.
- Pulso de Stock.
- Ficha de Decisión.
- Libro de Materiales.
- Semáforo Técnico.

## Lenguaje visual

- Fondo cálido tipo papel técnico.
- Negro grafito para estructura.
- Color del proyecto como acento contextual.
- Tipografía editorial en títulos y sans legible en controles.
- Texturas discretas, líneas de medición y amplias zonas de respiración.
- Pocos radios y sombras; evitar apariencia de plantilla SaaS.

## Responsive

- Escritorio: lienzo y ficha simultáneos.
- Tableta: panel lateral deslizable.
- Móvil: fotografía a pantalla completa y barra inferior por herramientas.

## Accesibilidad

- Objetivo WCAG 2.2 AA.
- Foco visible y teclado completo.
- Etiquetas persistentes.
- Estados con texto/iconos además de color.
- Movimiento reducido respetado.
