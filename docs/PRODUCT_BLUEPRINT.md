# Blueprint del producto

## Posicionamiento

La plataforma es un **estudio digital de proyectos**, no un e-commerce tradicional. El ambiente es la unidad visual y el proyecto es la entidad central.

## Cadena de valor

```text
Ambiente → medición → diagnóstico → color → producto → cantidad
→ materiales → stock → precio → cotización → pedido → ejecución
```

## Módulos y prioridad

| Módulo | Prioridad | Resultado |
|---|---:|---|
| Proyectos y ambientes | P0 | Contexto persistente por espacio |
| Calculadora de pintura | P0 | Área, manos, rendimiento y litros |
| Materiales complementarios | P0 | Preparación, aplicación y protección |
| Asistente de colores | P0 | Paletas explicadas y comprables |
| Catálogo contextual | P0 | Solo productos aptos para el proyecto |
| Inventario por sucursal | P0 | Disponibilidad real y trazable |
| Cotización versionada | P0 | PDF reproducible históricamente |
| Visualizador fotográfico | P1 | Comparación sobre imagen del usuario |
| Sustitución inteligente | P1 | Continuidad ante quiebres de stock |
| Modo profesional | P1 | Obras multiambiente, listas y plantillas |
| Proveedores/ERP/POS | P2 | Operación empresarial integrada |
| Escaneo espacial | P2 | Captura avanzada de dimensiones |

## Propuestas comerciales

Cada proyecto puede producir tres propuestas:

- **Económica:** cumple condiciones mínimas y prioriza inversión inicial.
- **Recomendada:** equilibrio de cobertura, lavabilidad, durabilidad y costo.
- **Premium:** maximiza desempeño, acabado o vida útil.

Cada alternativa recalcula litros y envases con su propia ficha técnica. No se permite cambiar solo el precio.

## Riesgos del dominio

- Confundir área de piso con paredes.
- Usar rendimientos promocionales como rendimiento real.
- Cotizar colores no reproducibles en la base disponible.
- Mostrar stock desactualizado.
- Sustituir por categoría sin validar compatibilidad.
- Modificar cotizaciones históricas al cambiar catálogo.
- Presentar una simulación de color como resultado exacto.
